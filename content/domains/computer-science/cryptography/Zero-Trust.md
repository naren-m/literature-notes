# Zero-Trust

Zero-trust security removes the assumption that something is trustworthy just because it is inside a network boundary.

The operating rule is "never trust, always verify." Each access attempt should be evaluated with identity, device state, context, requested resource, and least-privilege policy. This reduces the blast radius when an attacker obtains a credential or lands inside part of the environment.

Zero-trust is therefore less a product than an architecture discipline. It depends on [[Strong Authentication]], segmentation, continuous authorization, monitoring, and careful handling of [[Data Sensitivity]].
